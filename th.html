Prototype จริง

Claude Code คือ AI ผู้ช่วยที่อยู่ใน Terminal ของคุณ (หน้าต่างสีดำที่มีตัวหนังสือ) คุณพิมพ์คุยกับมันเป็นภาษาธรรมดา แล้วมันจะเขียนโค้ดให้คุณ นอกจากนี้มันยังอ่านงานออกแบบ Figma ของคุณได้โดยตรง

Expo คือเครื่องมือที่แปลงโค้ดเป็นแอป iOS จริง คุณไม่ต้องเขียนโค้ดเอง — Claude จะทำให้ คุณแค่บอกว่าต้องการอะไร

เมื่อทำตามคู่มือนี้จบ คุณจะมี Prototype iOS จริงที่ทำงานบน iPhone Simulator บน Mac ของคุณ

ติดตั้งครั้งเดียว

ส่วนนี้ทำแค่ครั้งเดียว หลังจากนี้การสร้าง Prototype ใหม่จะเร็วมาก

1

ติดตั้ง Xcode (สำหรับ iPhone Simulator)

Xcode คือเครื่องมือพัฒนาของ Apple คุณต้องใช้มันเพื่อรัน iPhone Simulator ไฟล์มีขนาดใหญ่ (~12 GB) ควรเริ่มดาวน์โหลดก่อน

เปิด App Store บน Mac ค้นหา Xcode แล้วกด Get รอให้ติดตั้งเสร็จ

เมื่อติดตั้งเสร็จแล้ว เปิด Xcode สักครั้งแล้วยอมรับ License จากนั้นเปิด Terminal แล้วรันคำสั่ง:

Terminal $ sudo xcode-select --install

มันจะถามรหัสผ่าน Mac ของคุณ (ที่ใช้ล็อกอิน) ขณะพิมพ์จะไม่มีอะไรขึ้นบนหน้าจอ — นั่นปกติ พิมพ์เสร็จแล้วกด Enter

Terminal คืออะไร? เปิด Spotlight (Cmd + Space) พิมพ์ Terminal แล้วกด Enter มันคือวิธีสั่งงาน Mac ผ่านข้อความ คุณจะวางคำสั่งลงไปในนี้ ถ้าคุณใช้ iTerm2 หรือ Ghostty อยู่แล้ว ก็ใช้ได้เหมือนกัน — ใช้ตัวไหนก็ได้ที่ถนัด
2

ติดตั้ง Homebrew

Homebrew เหมือน App Store สำหรับเครื่องมือนักพัฒนา แต่ทำงานใน Terminal คุณจะใช้มันติดตั้งของที่เหลือทั้งหมด

Terminal — วางบรรทัดนี้ทั้งหมด $ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

ทำตามคำแนะนำที่แสดง ตอนท้ายอาจมีข้อความ "Next steps" — ให้รันคำสั่งเหล่านั้นด้วย

มี Homebrew อยู่แล้ว? ถ้าเคยทำงานพัฒนามาก่อน อาจมีอยู่แล้ว ลองรัน brew --version — ถ้าแสดงเลขเวอร์ชัน ข้ามขั้นตอนนี้ได้เลย
3

ติดตั้ง Node.js

Node.js ใช้รัน JavaScript ที่ขับเคลื่อนแอปของคุณ ติดตั้งด้วย Homebrew:

Terminal $ brew install node

ตรวจสอบว่าใช้ได้:

Terminal $ node --version # ควรแสดงอะไรประมาณ v22.x.x
4

ติดตั้ง Claude Code

ติดตั้ง AI ผู้ช่วยที่คุณจะคุยด้วย รันคำสั่งนี้:

Terminal — วางบรรทัดนี้ทั้งหมด $ curl -fsSL https://claude.ai/install.sh | bash

ตรวจสอบว่าใช้ได้:

Terminal $ claude --version

ครั้งแรกที่รัน Claude มันจะขอให้คุณล็อกอินบัญชี Anthropic ทำตามขั้นตอน — มันจะเปิดหน้าเว็บในเบราว์เซอร์

อัปเดตอัตโนมัติ Claude Code อัปเดตตัวเองอัตโนมัติในพื้นหลัง คุณจะได้ใช้เวอร์ชันล่าสุดเสมอ ไม่ต้องรันคำสั่งอัปเดตเอง
5

เชื่อมต่อ Claude กับ Figma

ขั้นตอนนี้ให้ Claude อ่านงานออกแบบ Figma ของคุณได้โดยตรง รันคำสั่งนี้ใน Terminal (ไม่ใช่ใน Claude):

Terminal $ claude mcp add --transport http figma-remote-mcp https://mcp.figma.com/mcp

ครั้งแรกที่ Claude ใช้ Figma มันจะเปิดหน้าเว็บขอให้คุณอนุญาตการเชื่อมต่อ กด Allow แล้วเสร็จ

มันทำอะไร? คำสั่งนี้เพิ่ม Figma MCP (Model Context Protocol) server ให้ Claude หมายความว่า Claude จะดูข้างในไฟล์ Figma ของคุณได้ — อ่าน Frame, ตรวจสอบ Style, เข้าใจ Layout — แทนที่จะเดาจากภาพหน้าจอ ผลลัพธ์จะแม่นยำกว่ามาก
6

ติดตั้งเครื่องมือ iOS Simulator

เปิด Xcode แล้วไปที่ Xcode → Settings → Platforms ตรวจสอบว่าดาวน์โหลด iOS Simulator runtime แล้ว (เช่น iOS 18.x)

ตรวจสอบว่า Simulator ใช้ได้โดยรัน:

Terminal $ open -a Simulator

iPhone ควรปรากฏบนหน้าจอ ปิดไว้ก่อนได้

🎉

คุณเพิ่งเซ็ตอัป Dev Environment เสร็จแล้ว!

จริงๆ นะ — นี่คือสิ่งที่นักออกแบบส่วนใหญ่ไม่เคยทำ พักหายใจ ไปชงกาแฟสักแก้ว แล้วภูมิใจในตัวเองได้เลย ส่วนที่น่าเบื่อจบแล้ว ต่อจากนี้คือส่วนสนุก!

สร้าง Prototype แรกของคุณ

7

สร้างโฟลเดอร์โปรเจกต์

คิดว่าเหมือนสร้างโฟลเดอร์ใหม่ใน Finder รันคำสั่งเหล่านี้ทีละบรรทัด:

Terminal # สร้างโฟลเดอร์สำหรับ Prototype ทั้งหมด $ mkdir -p ~/Prototypes # เข้าไปในโฟลเดอร์นั้น $ cd ~/Prototypes
8

เริ่ม Claude Code

เปิด Claude ตัว flag พิเศษนี้จะให้ Claude รันคำสั่งเองได้โดยไม่ต้องขออนุญาตทุกครั้ง — ทำให้ใช้งานลื่นขึ้นมาก

Terminal $ claude --dangerously-skip-permissions
เกี่ยวกับชื่อ flag ที่ดูน่ากลัว --dangerously-skip-permissions แค่หมายความว่า Claude สามารถรันคำสั่ง Terminal ได้โดยไม่ต้องถามทุกครั้ง ปลอดภัยสำหรับการทำ Prototype Anthropic ตั้งชื่อแบบนี้เพื่อให้แน่ใจว่าคนใช้ตั้งใจเปิดจริงๆ

ตอนนี้คุณจะเห็นช่องให้พิมพ์ข้อความถึง Claude นี่คือพื้นที่ทำงานใหม่ของคุณ — คุณพูด Claude เขียนโค้ด

9

บอก Claude ให้สร้างแอป Expo

แค่คุยกับ Claude เป็นภาษาธรรมดา พิมพ์อะไรประมาณนี้:

คุณ → Claude Create a new Expo app called "my-prototype" with TypeScript. Set it up for iOS only. Use Expo Router for navigation. This is a design prototype — there is no backend or server. Mock all API calls with realistic sample data. Store any user data locally on the device only (e.g. AsyncStorage). Everything should work offline with no external dependencies. Then run it in the iOS simulator so I can see it.
ทำไมต้อง "mock" และ "no backend"? แอปจริงจะคุยกับเซิร์ฟเวอร์เพื่อดึงข้อมูล สำหรับ Prototype ไม่ต้องทำแบบนั้น — Claude จะสร้างข้อมูลปลอมที่ดูสมจริงอยู่ในตัวแอปเลย ทุกอย่างทำงานในเครื่อง ไม่ต้องใช้อินเทอร์เน็ต และไม่ต้องตั้งเซิร์ฟเวอร์หรือฐานข้อมูลใดๆ

Claude จะ:

  • สร้างโปรเจกต์พร้อมการตั้งค่าที่ถูกต้อง
  • ติดตั้ง Dependencies
  • ตั้งค่าสำหรับ iOS
  • เปิด iOS Simulator พร้อมแอปของคุณ

ครั้งแรกจะใช้เวลาสักครู่ คุณจะเห็น iPhone Simulator โผล่ขึ้นมาพร้อมแอปของคุณ

คุณเพิ่งสร้างแอปได้แล้ว! ถ้าเห็นอะไรบน Simulator ยินดีด้วย — คุณมีแอป iOS ที่ทำงานจริงแล้ว ต่อจากนี้แค่ทำให้มันดูเหมือนงานออกแบบของคุณ

เปลี่ยนงานออกแบบ Figma เป็นหน้าจอ

10

สร้างหน้าจอจาก Figma

ตอนนี้ Claude เชื่อมต่อกับ Figma แล้ว (ขั้นตอนที่ 5) คุณสามารถส่งงานออกแบบให้ Claude ได้เลย:

a) ใน Figma เปิดไฟล์ที่มีงานออกแบบ คลิกขวาที่ Frame ที่ต้องการสร้าง แล้วเลือก Copy link

b) กลับไปที่ Claude แล้วพิมพ์อะไรประมาณนี้:

คุณ → Claude Here's my Figma design for the home screen: https://www.figma.com/design/ABC123/My-App?node-id=1-234 Build this as the home screen of the app. Match the design as closely as possible — colors, spacing, typography, layout. Use realistic mock data for any content. No real API calls — keep everything local and working offline.

เพราะคุณตั้งค่าการเชื่อมต่อ Figma ไว้แล้วในขั้นตอนที่ 5 Claude จะอ่านงานออกแบบของคุณได้โดยตรง — ตรวจสอบสี, Spacing, Typography และ Layout แล้วเขียนโค้ด React Native ให้ตรงกัน iOS Simulator จะรีโหลดอัตโนมัติพร้อมหน้าจอใหม่

11

ปรับแต่งงานออกแบบ

ตรงนี้แหละที่สนุก แค่คุยกับ Claude ต่อ:

ตัวอย่างการสนทนา # ปรับแต่งดีไซน์ "The header should be sticky when scrolling" # เพิ่มหน้าจอใหม่ "Now build the profile screen from this Figma frame: https://www.figma.com/design/ABC123/My-App?node-id=5-678" # เพิ่มการนำทางระหว่างหน้าจอ "When I tap the profile icon in the tab bar, navigate to the profile screen" # เพิ่ม Interaction "Add a pull-to-refresh animation on the feed screen" # แก้ไขอะไรบางอย่าง "The spacing between the cards is too tight, add more breathing room"

ทุกครั้งที่สั่ง Claude จะอัปเดตโค้ดแล้ว Simulator จะรีเฟรชอัตโนมัติ เหมือนมี Developer นั่งอยู่ข้างๆ คุณ

เคล็ดลับเพื่อผลลัพธ์ที่ดี

→

บอกให้ชัดเจนว่าต้องการอะไร

แทนที่จะบอกว่า "make it look better" ลองบอกว่า "increase the padding around the card to 24px and change the title to 20px semibold" ยิ่งบอกชัดเจน Claude ยิ่งทำได้ตรงใจ

→

ทำทีละหน้าจอ

อย่าพยายามสร้างทั้งแอปในคราวเดียว เริ่มจากหน้าจอเดียว ทำให้ถูกใจ แล้วค่อยไปหน้าถัดไป วิธีนี้ให้ Context กับ Claude ได้ดีกว่าและผลลัพธ์จะดีกว่า

→

แชร์ Frame แต่ละอันจาก Figma

เวลา Copy ลิงก์ Figma ให้เลือก Frame เฉพาะ (เช่น หน้าจอเดียว) ไม่ใช่ทั้งหน้า จะช่วยให้ Claude โฟกัสได้ตรงจุดที่ต้องการ

→

ใช้ศัพท์ดีไซน์ได้เลย

Claude เข้าใจภาษาดีไซน์ พูดอะไรแบบ "add a bottom sheet", "use a skeleton loader", "make this a horizontal scroll with snap points" หรือ "add a parallax effect to the hero image" ได้เลย

→

บอก Claude ให้เริ่มใหม่ได้

ถ้าหน้าจอไหนไม่ถูกใจ บอกได้เลยว่า "let's scrap the profile screen and rebuild it from scratch" Claude ไม่ว่าหรอก

โบนัส: สร้าง Static Page แล้วเผยแพร่

ไม่จำเป็นต้องใช้ Expo ทุกครั้ง Claude สร้างหน้า HTML แบบ Standalone ได้ด้วย — หน้า Landing Page, Case Study, Portfolio, เอกสารงานออกแบบ — แล้วเผยแพร่บนเว็บฟรีด้วย GitHub Pages จริงๆ แล้วหน้าที่คุณกำลังอ่านนี้ก็สร้างแบบนี้เหมือนกัน

12

ติดตั้ง GitHub CLI

GitHub คือที่ที่หน้าเว็บของคุณจะถูกโฮสต์ CLI (command-line interface) ให้ Claude สร้าง Repo และ Deploy หน้าเว็บได้โดยไม่ต้องออกจาก Terminal

Terminal $ brew install gh

จากนั้นล็อกอินบัญชี GitHub:

Terminal $ gh auth login

มันจะถามคำถามไม่กี่ข้อ เลือก:

  • GitHub.com (ไม่ใช่ Enterprise)
  • HTTPS เป็น Protocol
  • Login with a web browser เมื่อถามวิธียืนยันตัวตน

ทำตามขั้นตอน — มันจะเปิดเบราว์เซอร์เพื่ออนุญาตการเชื่อมต่อ

ยังไม่มีบัญชี GitHub? ไปที่ github.com แล้วสมัครบัญชีฟรี ใช้เวลาแค่ 2 นาที คุณต้องใช้มันเพื่อเผยแพร่หน้าเว็บ
13

สร้างโปรเจกต์ Static Page

สร้างโฟลเดอร์ใหม่แล้วเริ่ม Claude:

Terminal # สร้างโฟลเดอร์สำหรับหน้าเว็บ $ mkdir -p ~/Prototypes/my-page $ cd ~/Prototypes/my-page # เริ่ม Claude $ claude --dangerously-skip-permissions
14

บอก Claude ว่าจะสร้างอะไร

อธิบายหน้าเว็บที่ต้องการ Claude จะสร้างไฟล์ HTML ที่มี Style และ Interaction ครบในตัว นี่คือตัวอย่าง:

คุณ → Claude — ตัวอย่าง: หน้า Portfolio Build a single-page portfolio site for a UI designer. Use a minimal, editorial style with beautiful typography. Include sections for: hero with name and tagline, selected work (use placeholder images from unsplash), about, and contact. Make it responsive. Everything in a single index.html file with inline CSS and no external dependencies.
คุณ → Claude — ตัวอย่าง: Case Study Build a case study page for a mobile app redesign project. Include: project overview, problem statement, process with before/after comparisons, key screens (use placeholder images), outcomes and metrics. Clean, professional layout. Single index.html file, no dependencies.
คุณ → Claude — ตัวอย่าง: จาก Figma Here's my Figma design for a landing page: https://www.figma.com/design/ABC123/Landing?node-id=1-234 Build this as a static HTML page. Match the design exactly. Single index.html file with inline CSS. Make it responsive. No external dependencies.

Claude จะสร้างไฟล์ HTML ให้ ดูตัวอย่างได้โดยพิมพ์:

คุณ → Claude Open the page in my browser so I can see it.

ปรับแต่งต่อได้เรื่อยๆ — "make the hero bigger", "add a dark mode toggle", "change the font to something more playful" — จนกว่าจะพอใจ

15

เผยแพร่บนเว็บด้วย GitHub Pages

เมื่อพอใจกับหน้าเว็บแล้ว แค่บอก Claude ให้เผยแพร่:

คุณ → Claude Create a GitHub repository called "my-page", push this project to it, and set up GitHub Pages so it's live on the web. Give me the URL when it's done.

Claude จะ:

  • สร้าง Git Repository
  • สร้าง Repo บนบัญชี GitHub ของคุณ
  • Push โค้ดขึ้นไป
  • เปิดใช้งาน GitHub Pages
  • ให้ URL ที่เข้าถึงได้

หน้าเว็บจะเข้าถึงได้ที่ https://yourusername.github.io/my-page/ ภายในไม่กี่นาที แชร์ลิงก์นี้ให้ใครก็ได้

โฮสต์ฟรี ตลอดไป GitHub Pages ฟรีสำหรับ Repo สาธารณะ หน้าเว็บจะออนไลน์อยู่ตลอดเท่าที่คุณเก็บ Repository ไว้ ไม่มีค่าโฮสต์ ไม่ต้องจัดการเซิร์ฟเวอร์ ไม่มีลิงก์หมดอายุ
16

อัปเดตหน้าเว็บเมื่อไหร่ก็ได้

อยากแก้ไขทีหลัง? แค่เปิดโปรเจกต์แล้วบอก Claude:

Terminal $ cd ~/Prototypes/my-page $ claude --dangerously-skip-permissions
คุณ → Claude Update the hero section with a new tagline: "Designing experiences that matter." Then push the changes to GitHub so the live page updates.

Claude จะแก้ไขไฟล์ Commit การเปลี่ยนแปลง แล้ว Push ขึ้นไป หน้าเว็บจะอัปเดตอัตโนมัติภายในไม่กี่นาที

คำถามที่พบบ่อย

ต้องเขียนโค้ดเป็นไหม?

ไม่ต้อง Claude เขียนโค้ดทั้งหมด คุณแค่อธิบายสิ่งที่ต้องการเป็นภาษาธรรมดาและอ้างอิงงานออกแบบ Figma คิดว่าเหมือนกำกับ Developer — คุณให้วิสัยทัศน์ Claude จัดการ Implementation

จะตรงกับงานออกแบบ Figma แค่ไหน?

ใกล้เคียงมากสำหรับ Layout, สี, Typography และ Spacing แอนิเมชันซับซ้อนหรือภาพวาดพิเศษอาจต้องปรับไปมาบ้าง ยิ่งเลือก Frame ที่เฉพาะเจาะจงจาก Figma ยิ่งตรง

ทดสอบบน iPhone จริงได้ไหม?

ได้! ถาม Claude ว่า: "How do I run this on my real iPhone?" มันจะแนะนำขั้นตอน ต้องเชื่อมต่อ iPhone ด้วย USB และมีบัญชี Apple Developer (ฟรีก็ใช้ทดสอบได้)

ถ้า Claude ติดหรือแอปพัง?

แค่อธิบายสิ่งที่เห็น เช่น: "The app crashed and I see a red error screen that says TypeError" หรือ "The simulator is just showing a white screen" Claude มักจะหาทางแก้ได้

ปิด Terminal แล้วกลับมาทีหลังได้ไหม?

ได้ ไฟล์โปรเจกต์จะอยู่ใน Mac คุณ แค่เปิด Terminal, cd ไปที่โฟลเดอร์โปรเจกต์ แล้วเริ่ม Claude ใหม่ โค้ดยังอยู่ — Claude แค่เริ่มการสนทนาใหม่ บอกว่า "look at the existing code and tell me what screens we have so far" เพื่อดูว่ามีอะไรบ้าง

Claude เพิ่มข้อมูลจริงหรือเชื่อมต่อ API ได้ไหม?

สำหรับ Prototyping ไม่ต้องใช้ API จริงหรือ Backend Claude จะสร้างข้อมูล Mock ที่ดูสมจริงและเก็บทุกอย่างในเครื่อง ถ้าหน้าจอแสดงฟีด, โปรไฟล์, หรือข้อความ Claude จะสร้างข้อมูลตัวอย่างที่น่าเชื่อถือ แค่บอกว่า "use realistic sample data for the feed" ก็พอ

แชร์ Prototype ให้คนอื่นยังไง?

วิธีง่ายสุดคืออัดหน้าจอ — ใช้ Cmd+Shift+5 บน macOS เพื่ออัด Simulator ถ้าต้องการแชร์ Build ที่คนอื่นติดตั้งได้ ถาม Claude เกี่ยวกับ EAS Build ของ Expo หรือ TestFlight

ถ้าอยากเริ่ม Prototype ใหม่เลย?

กลับไปขั้นตอนที่ 7 สร้างโฟลเดอร์ใหม่ใน ~/Prototypes/ แล้วเริ่มใหม่

สร้างสำหรับนักออกแบบที่ต้องการทำ Prototype เร็วขึ้น ขับเคลื่อนโดย Claude Code และ Expo
