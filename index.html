A guide for designers

Turn your Figma designs into real, working prototypes

Use Claude Code + Expo to build native iOS apps from your designs. No coding experience needed — just your Mac, your Figma file, and a conversation.

Claude Code is an AI assistant that lives in your terminal (that black window with text). You talk to it in plain English, and it writes code for you. It can also read your Figma designs directly.

Expo is a tool that turns code into native iOS apps. You won't need to write the code yourself — Claude does that. You just tell it what you want.

By the end of this guide you'll have a real iOS prototype running on the iPhone simulator on your Mac.

One-time setup

This section only needs to be done once. After that, creating new prototypes is fast.

1

Install Xcode (for the iPhone simulator)

Xcode is Apple's development tool. You need it to run the iPhone simulator. It's a large download (~12 GB), so start this first.

Open the App Store on your Mac, search for Xcode, and click Get. Wait for it to finish installing.

Once installed, open Xcode once and agree to the license. Then open Terminal and run:

Terminal $ sudo xcode-select --install

It will ask for your Mac password (the one you use to log in). As you type, nothing will appear on screen — that's normal. Press Enter when done.

What's the Terminal? Open Spotlight (Cmd + Space), type Terminal, and press Enter. It's a text-based way to talk to your Mac. You'll paste commands into it. If you already use iTerm2 or Ghostty, those work just as well — use whichever you prefer.
2

Install Homebrew

Homebrew is like an app store for developer tools, but it runs in the terminal. You'll use it to install everything else.

Terminal — paste this entire line $ /bin/bash -c "$(curl -fsSL https://raw.githubusercontent.com/Homebrew/install/HEAD/install.sh)"

Follow the instructions it prints. At the end it may say "Next steps" — run those commands too.

Already have Homebrew? If you've done development work before, you might already have it. Run brew --version — if it prints a version number, skip this step.
3

Install Node.js

Node.js runs the JavaScript that powers your app. Install it with Homebrew:

Terminal $ brew install node

Verify it works:

Terminal $ node --version # Should print something like v22.x.x
4

Install Claude Code

This installs the AI assistant you'll be talking to. Run this single command:

Terminal — paste this entire line $ curl -fsSL https://claude.ai/install.sh | bash

Verify it works:

Terminal $ claude --version

The first time you run Claude, it will ask you to log in to your Anthropic account. Follow the prompts — it will open a browser window.

Automatic updates Claude Code updates itself automatically in the background, so you'll always have the latest version. No need to run any update commands.
5

Connect Claude to Figma

This lets Claude read your Figma designs directly. Run this command in Terminal (not inside Claude):

Terminal $ claude mcp add --transport http figma-remote-mcp https://mcp.figma.com/mcp

The first time Claude uses Figma, it will open a browser window asking you to authorize the connection. Click Allow and you're done.

What does this do? This adds the Figma MCP (Model Context Protocol) server to Claude. It means Claude can look inside your Figma files — read frames, inspect styles, understand your layouts — instead of just guessing from a screenshot. Much more accurate results.
6

Install iOS simulator tools

Open Xcode, then go to Xcode → Settings → Platforms and make sure an iOS Simulator runtime is downloaded (e.g. iOS 18.x).

You can verify the simulator works by running:

Terminal $ open -a Simulator

An iPhone should appear on your screen. You can close it for now.

🎉

You just set up a dev environment.

Seriously — that's something most designers never do. Take a breath, grab a coffee, feel smug. The boring part is over. Everything from here is the fun part.

Creating your first prototype

7

Create a project folder

Think of this as creating a new folder for your project, just like you would in Finder. Run these commands one at a time:

Terminal # Create a folder for all your prototypes $ mkdir -p ~/Prototypes # Go into that folder $ cd ~/Prototypes
8

Start Claude Code

Now launch Claude. The special flag lets Claude run commands on its own without asking you for permission each time — this makes the experience much smoother.

Terminal $ claude --dangerously-skip-permissions
About that scary flag name The --dangerously-skip-permissions flag just means Claude can run terminal commands without asking you each time. It's safe for prototyping. Anthropic named it that way to make sure people only use it intentionally.

You should now see a prompt where you can type messages to Claude. This is your new workspace — you talk, Claude codes.

9

Tell Claude to create an Expo app

Now just talk to Claude in plain English. Type something like this:

You → Claude Create a new Expo app called "my-prototype" with TypeScript. Set it up for iOS only. Use Expo Router for navigation. This is a design prototype — there is no backend or server. Mock all API calls with realistic sample data. Store any user data locally on the device only (e.g. AsyncStorage). Everything should work offline with no external dependencies. Then run it in the iOS simulator so I can see it.
Why "mock" and "no backend"? A real app talks to servers to fetch data. For a prototype, you don't need that — Claude will generate realistic fake data that lives right inside the app. Everything runs locally, nothing needs the internet, and you won't need to set up any servers or databases.

Claude will:

  • Create the project with all the right settings
  • Install dependencies
  • Configure it for iOS
  • Launch the iOS simulator with your app running

This takes a few minutes the first time. You'll see an iPhone simulator pop up with your app.

You just built an app If you can see something on the simulator, congratulations — you have a running iOS app. Everything from here is just making it look like your design.

Turning your Figma designs into screens

10

Build screens from Figma

Now that Claude is connected to Figma (step 5), you can feed it your designs directly:

a) In Figma, open the file with your designs. Right-click the frame you want to build and choose Copy link.

b) Go back to Claude and type something like:

You → Claude Here's my Figma design for the home screen: https://www.figma.com/design/ABC123/My-App?node-id=1-234 Build this as the home screen of the app. Match the design as closely as possible — colors, spacing, typography, layout. Use realistic mock data for any content. No real API calls — keep everything local and working offline.

Because you set up the Figma connection in step 5, Claude can read your design directly — inspecting colors, spacing, typography, and layout. It will write the React Native code to match. The iOS simulator will automatically reload with the new screen.

11

Iterate on the design

Here's where it gets fun. Just keep talking to Claude:

Example conversations # Adjust the design "The header should be sticky when scrolling" # Add a new screen "Now build the profile screen from this Figma frame: https://www.figma.com/design/ABC123/My-App?node-id=5-678" # Add navigation between screens "When I tap the profile icon in the tab bar, navigate to the profile screen" # Add interactions "Add a pull-to-refresh animation on the feed screen" # Fix something "The spacing between the cards is too tight, add more breathing room"

Each time, Claude will update the code and the simulator will refresh automatically. It's like having a conversation with a developer sitting next to you.

Tips for great results

→

Be specific about what you want

Instead of "make it look better", try "increase the padding around the card to 24px and change the title to 20px semibold." The more precise you are, the better Claude can match your intent.

→

Work screen by screen

Don't try to build the entire app at once. Start with one screen, get it right, then move to the next. This gives Claude better context and produces better results.

→

Share individual frames from Figma

When you copy a Figma link, make sure you've selected a specific frame (like a single screen), not the entire page. This helps Claude focus on exactly what you want.

→

Use design vocabulary

Claude understands design language. Feel free to say things like "add a bottom sheet", "use a skeleton loader", "make this a horizontal scroll with snap points", or "add a parallax effect to the hero image".

→

Ask Claude to start over if needed

If a screen isn't working out, it's totally fine to say "let's scrap the profile screen and rebuild it from scratch." Claude won't mind.

→

Share your prototype with others via tunnel

Want someone else to see your prototype on their phone — without building or deploying anything? Expo can create a public tunnel that lets anyone open your app via a link.

First, install the tunneling tool (one time only):

Terminal $ npm i -g @expo/ngrok

Then tell Claude to start the app with tunneling:

You → Claude Start the app with tunnel mode so I can share it with someone on another device.

Claude will run npx expo start --tunnel, which generates a public URL. The other person needs the Expo Go app on their phone (free on the App Store) — they scan the QR code or open the URL, and your prototype loads on their device.

Good to know Tunneling is slower than the simulator since it routes through the internet. It works best for quick demos, not all-day testing. Your Mac needs to stay on and running the tunnel while others are viewing.

Slash commands

While you're chatting with Claude, you can type special commands that start with /. These aren't messages to Claude — they're controls for the tool itself.

/

/status — Check your usage

Type /status to see how much of your usage allowance you've consumed. Claude has rate limits — if you're doing a lot of work, you may hit them.

Hitting rate limits? If /status shows you're near your limit, or Claude starts responding slower or refuses to answer, don't panic. Ask for help on Slack — we can sort out your usage or give you a different API key.
/

/clear — Start a fresh conversation

If Claude seems confused or is referencing things that no longer exist, type /clear to reset the conversation. Your project files stay untouched — only the chat history is cleared.

/

/help — See all available commands

Type /help to see the full list of slash commands. You won't need most of them, but it's good to know it's there.

/

/btw — Ask a quick side question

Need a quick answer while Claude is working? Type /btw followed by your question. It pops up in an overlay, answers from what Claude already knows about your conversation, and disappears when you press Esc. It doesn't interrupt the main task and doesn't add to your conversation history.

When to use /btw vs. a normal message Use /btw for quick factual lookups from the current session — "what was that file called?" or "what port is the dev server on?". If you need Claude to read files, run commands, or have a back-and-forth discussion, just type a normal message instead.
⎋

Esc Esc — Undo and go back

Made a wrong turn? Press Esc twice to open the rewind menu. You'll see a list of every prompt you've sent — pick the point you want to go back to, and Claude will undo both the conversation and the code changes it made after that point.

This is your safety net. If Claude took your design in the wrong direction, broke something, or you just want to try a different approach — rewind and try again. Your code goes back to exactly how it was.

Don't be afraid to experiment Because you can always rewind, you can try bold ideas without risk. Ask Claude to completely redesign a screen, try a different navigation pattern, or experiment with animations. If you don't like the result, Esc Esc and you're right back where you started.

Bonus: Build static pages and publish them

You don't need Expo for everything. Claude can also build standalone HTML pages — landing pages, case studies, portfolios, design documentation — and publish them to the web for free using GitHub Pages. This is actually how the page you're reading right now was made.

12

Install GitHub CLI

GitHub is where your page will be hosted. The CLI (command-line interface) lets Claude create repos and deploy pages for you without leaving the terminal.

Terminal $ brew install gh

Then log in to your GitHub account:

Terminal $ gh auth login

It will ask you a few questions. Choose:

  • GitHub.com (not Enterprise)
  • HTTPS as the protocol
  • Login with a web browser when asked how to authenticate

Follow the prompts — it will open your browser to authorize the connection.

Don't have a GitHub account? Go to github.com and sign up for a free account first. It takes 2 minutes. You'll need it to publish your pages.
13

Create a static page project

Create a new folder and start Claude in it:

Terminal # Create a folder for your page $ mkdir -p ~/Prototypes/my-page $ cd ~/Prototypes/my-page # Start Claude $ claude --dangerously-skip-permissions
14

Tell Claude what to build

Describe the page you want. Claude will create a self-contained HTML file with all styles and interactions built in. Here are some ideas:

You → Claude — example: portfolio page Build a single-page portfolio site for a UI designer. Use a minimal, editorial style with beautiful typography. Include sections for: hero with name and tagline, selected work (use placeholder images from unsplash), about, and contact. Make it responsive. Everything in a single index.html file with inline CSS and no external dependencies.
You → Claude — example: case study Build a case study page for a mobile app redesign project. Include: project overview, problem statement, process with before/after comparisons, key screens (use placeholder images), outcomes and metrics. Clean, professional layout. Single index.html file, no dependencies.
You → Claude — example: from Figma Here's my Figma design for a landing page: https://www.figma.com/design/ABC123/Landing?node-id=1-234 Build this as a static HTML page. Match the design exactly. Single index.html file with inline CSS. Make it responsive. No external dependencies.

Claude will create the HTML file. You can preview it by asking:

You → Claude Open the page in my browser so I can see it.

Keep iterating — "make the hero bigger", "add a dark mode toggle", "change the font to something more playful" — until you're happy with it.

15

Publish it to the web with GitHub Pages

Once you're happy with the page, just tell Claude to publish it:

You → Claude Create a GitHub repository called "my-page", push this project to it, and set up GitHub Pages so it's live on the web. Give me the URL when it's done.

Claude will:

  • Initialize a Git repository
  • Create a repo on your GitHub account
  • Push the code
  • Enable GitHub Pages
  • Give you the live URL

Your page will be live at https://yourusername.github.io/my-page/ within a minute or two. Share that link with anyone.

Free hosting, forever GitHub Pages is free for public repos. Your page will stay online as long as you keep the repository. No hosting fees, no server management, no expiring links.
16

Update your page anytime

Want to make changes later? Just open the project and tell Claude what to change:

Terminal $ cd ~/Prototypes/my-page $ claude --dangerously-skip-permissions
You → Claude Update the hero section with a new tagline: "Designing experiences that matter." Then push the changes to GitHub so the live page updates.

Claude will edit the file, commit the change, and push it. The live page updates automatically within a minute.

Common questions

Do I need to know how to code?

No. Claude writes all the code. You describe what you want in plain English and reference your Figma designs. Think of it like directing a developer — you provide the vision, Claude handles the implementation.

How close will it match my Figma design?

Very close for layout, colors, typography, and spacing. Complex animations or custom illustrations may need some back-and-forth. The more specific your Figma link (select the exact frame), the better the match.

Can I test on a real iPhone?

Yes! Ask Claude: "How do I run this on my real iPhone?" It will walk you through it. You'll need to connect your iPhone via USB and have an Apple Developer account (free works for testing).

What if Claude gets stuck or the app crashes?

Just describe what you see. Say something like: "The app crashed and I see a red error screen that says TypeError" or "The simulator is just showing a white screen." Claude can usually figure it out and fix it.

Expo Go says it needs a newer version, but there's no update in the App Store?

This happens when your project uses a newer Expo SDK (e.g. SDK 55) than the version of Expo Go available in the App Store. Install the latest Expo Go beta via TestFlight: testflight.apple.com/join/GZJxxfUU. The TestFlight version supports the newest SDK before it hits the App Store.

Can I close the terminal and come back later?

Yes. The project files stay on your Mac. Just open Terminal, cd into your project folder, and start Claude again. Your code is still there — Claude just starts a new conversation. You can tell it "look at the existing code and tell me what screens we have so far" to get back up to speed.

Can Claude add real data or API connections?

For prototyping, you don't need real APIs or a backend. Claude generates realistic mock data and stores everything locally on the device. If a screen shows a feed, user profiles, or messages, Claude will create convincing sample data that makes the prototype feel real. Just say "use realistic sample data for the feed" and it will.

How do I share the prototype with others?

Three options: Screen recording — use Cmd+Shift+5 on macOS to record the simulator. Expo tunnel — run npx expo start --tunnel (see the tip above) to let others open your prototype on their phone in real time via Expo Go. Installable build — ask Claude about Expo's EAS Build or TestFlight for a build others can install without your Mac running.

What if I want to start a completely new prototype?

Go back to step 7. Create a new folder inside ~/Prototypes/ and start fresh.
